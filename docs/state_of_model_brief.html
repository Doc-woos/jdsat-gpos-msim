What Is Real Now

The app has moved beyond a toy MVP. It now has a stable shared-domain seam, reusable synthetic force packs, grouped analyst reporting, and portable export artifacts.

Implemented

  • Deterministic projection and baseline-vs-variant comparison.
  • Shared neutral manpower algorithms through `gameplan.domains.manpower`.
  • Named scenarios, decomposed manifests, and reusable synthetic force packs.
  • Grouped rollups by `occfld`, `community`, and `force_element`.
  • Grouped authorization/fill summaries and readiness pressure signals.
  • Full and compact exports that preserve grouped analyst context.

Not Implemented Yet

  • Billet-level structures, unit-position mapping, or billet fill by position.
  • Training pipeline throughput and washout modeling.
  • Monte Carlo, sensitivity, or optimization workflows.
  • Officer, warrant, or cross-service generalization.
  • Operational and budget integration.

Current Model Semantics

The right way to describe the present state is important because the visuals point beyond what is actually implemented.

Current Contract

  1. The model projects inventory and compares baseline vs variant scenarios.
  2. Pack-backed scenarios add group context for `occfld`, `community`, and `force_element`.
  3. That group context enables grouped inventory/demand rollups, grouped fill summaries, and grouped readiness pressure signals.
  4. Authorization artifacts are preferred when present; demand can act as a fallback proxy.
  5. The API and exports expose `authorization_basis` so analysts can see which semantic basis was used.
Important Constraint

This is a grouped readiness proxy only. It does not yet map inventory to explicit billets, positions, or unit-level vacancy structures.

That distinction should remain explicit in every external brief and demo until billet-level inputs actually exist.

Recommended Next Move

The next high-value work is still analyst-facing depth, not another data-model jump.

Recommendation

  1. Keep billet-level modeling out for now.
  2. Treat grouped authorization, fill, and readiness semantics as the active analyst contract.
  3. Improve explanation, reporting, and handoff artifacts on top of that contract.
  4. Only then decide whether a billet-level schema is concrete enough to lock in.
